1. Perbedaan Tanggung Jawab Front-End dan Back-End

Front-End bertanggung jawab pada tampilan dan interaksi pengguna, seperti halaman web, 
form input, tombol, dan desain antarmuka. Front-End berfungsi agar pengguna dapat berinteraksi 
dengan sistem secara nyaman.

Back-End bertanggung jawab pada proses pengolahan data, logika aplikasi, validasi input, 
serta pengiriman respon dari server ke pengguna.

Analogi toko:
Front-End diibaratkan sebagai etalase dan kasir, tempat pembeli melihat barang dan menyampaikan pesanan.  
Back-End diibaratkan sebagai dapur dan gudang, tempat pesanan diproses, dicek kebenarannya, 
dan disiapkan sebelum diberikan ke pembeli.

--------------------------------------------------

2. Perbedaan Method GET dan POST

Method GET digunakan untuk mengirim data melalui URL sehingga data dapat terlihat oleh pengguna.
GET biasanya digunakan untuk permintaan data sederhana seperti pencarian.

Method POST digunakan untuk mengirim data melalui body request sehingga data tidak terlihat 
di URL dan lebih aman. POST juga dapat mengirim data dengan ukuran lebih besar.

Method POST sebaiknya digunakan saat mengirim data penting seperti form kontak, login, 
atau data pribadi pengguna.

--------------------------------------------------

3. Alasan Validasi Input Harus Dilakukan di Server (Back-End)

Validasi input harus dilakukan di server karena validasi di browser (Front-End) dapat dimanipulasi atau dilewati oleh pengguna.  
Back-End berfungsi sebagai lapisan keamanan terakhir untuk memastikan data yang diterima sesuai aturan, aman, dan tidak merusak sistem.

Dengan melakukan validasi di server, aplikasi menjadi lebih aman dan data yang diproses lebih terjamin kebenarannya.
